Anywork365

One-page evidence-backed summary generated from the repository only.

What It Is

Anywork365 is a Next.js marketplace MVP for Nigeria that connects clients with artisans, technicians, engineers, and other service professionals. The repo includes discovery flows, job posting, authenticated dashboards, profile management, wallet operations, and payment hooks.

Who It's For

Primary persona: Nigerian clients who need to find and hire skilled service professionals. The app also has dedicated signup, profile, and withdrawal flows for professionals.

What It Does

  • Lists professionals with search, city filters, skill/category filters, and detail pages.
  • Lists jobs with search/filtering, detail views, and a protected job-posting flow.
  • Supports signup/login for client and professional roles.
  • Stores sessions in an httpOnly JWT cookie and protects dashboard routes with middleware.
  • Lets users update profile data, skills, city, bio, and upload avatar images.
  • Shows wallet balances, escrow, transaction history, and bank account status.
  • Integrates Paystack flows for funding, bank verification, payment verification, and withdrawals.

How It Works

  • UI: Next.js App Router pages in src/app render landing, listings, auth, and dashboard screens with shared React/Tailwind components.
  • API: Route handlers in src/app/api validate requests, read session state, and return JSON for auth, users, jobs, uploads, and wallet actions.
  • Auth: src/lib/auth.ts signs HS256 JWTs into the aw365_session cookie; src/middleware.ts redirects unauthenticated dashboard access.
  • Data: The repo uses mock arrays plus in-memory stores/maps for users, jobs, wallets, transactions, and withdrawals. Persistent database implementation: Not found in repo.
  • Payments + files: Paystack server utilities manage bank lookup, payment init/verify, and transfers; avatar uploads are written to public/uploads.

How To Run

  • cd anywork365
  • npm install
  • Copy .env.example to .env.local and keep at least local app/JWT values. Paystack features need valid Paystack keys.
  • npm run dev
  • Open http://localhost:3000
  • Demo login in repo: demo@anywork365.com / Demo1234
